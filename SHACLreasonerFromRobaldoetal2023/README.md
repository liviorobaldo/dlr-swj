<p>
  This folder contains the SHACL-based reasoner presented in
  <a href="https://link.springer.com/article/10.1007/s10506-023-09360-z">[Robaldo et al., 2023]</a>, adapted to measure performance on the same
  sample ABoxes used to evaluate the DLR reasoner.
</p>

<p>
  <a href="https://link.springer.com/article/10.1007/s10506-023-09360-z">[Robaldo et al., 2023]</a> surveys and compares seven of the most widely
  used legal reasoners in contemporary LegalTech research. The SHACL-based reasoner is among those exhibiting the best overall performance and is
  the only one, besides the DLR reasoner introduced in this paper, that can natively process RDF data.
</p>

<p>
  The comparison between the SHACL-based reasoner and the DLR reasoner shows that the latter consistently outperforms the former. The primary reason
  is that the DLR reasoner's rules are implemented in SPARQL, which requires less computational effort than the corresponding SHACL rules.
</p>

<p>
  The file <code>SHACLreasonerFromRobaldoetal2023.java</code> can be executed via
  <code>runSHACLreasonerFromRobaldoetal2023.bat</code>. It takes as input the SHACL rule files
  <code>DLRandDKBrules.ttl</code> and <code>DLRrulesCompliance.ttl</code>, together with the TBox
  <code>TBoxDLRandDKB.ttl</code>, executes the rules on all files contained in the <code>UseCases</code> subfolder,
  and reports the measured performance for each file. The files <code>DLRandDKBrules.ttl</code> and
  <code>TBoxDLRandDKB.ttl</code> are automatically generated by <code>runDRLrulesBuilder.bat</code>, while the
  <code>UseCases</code> subfolder is populated by <code>runGenerateAndRunRandomABoxes.bat</code>.
</p>
</div>
